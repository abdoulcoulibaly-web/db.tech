Les bases de données
modernes

Les bases de données sont utilisées pour stocker et organiser les informations. Elles sont essentielles dans les applications web et mobiles. Découvrez les trois piliers du stockage actuel.

Découvrir
MySQL • SQLite • MongoDB
Structure Relationnelle

MySQL

MySQL est utilisé dans la majorité des sites web pour stocker des données structurées comme les utilisateurs, les commandes ou les produits. Les données sont organisées dans des tables reliées entre elles.

Fonctionnement :

  • Les données sont stockées dans des tables composées de lignes et colonnes.
  • Chaque table possède une clé primaire qui identifie chaque ligne.
  • Les tables peuvent être liées entre elles avec des relations.
  • Les données sont manipulées avec le langage SQL.

Commandes SQL importantes :

  • SELECT → lire les données
  • INSERT → ajouter des données
  • UPDATE → modifier les données
  • DELETE → supprimer des données

Exemples réels :

  • Système de connexion utilisateur
  • Sites e-commerce
  • Gestion scolaire
  • Applications bancaires

Avantages :

  • Données bien organisées
  • Sécurité des transactions
  • Très utilisé dans le web

SQLite

SQLite est une base de données embarquée utilisée dans les applications mobiles et les logiciels locaux. Elle fonctionne sans serveur et stocke toutes les données dans un seul fichier.

Fonctionnement :

  • La base de données est contenue dans un seul fichier.
  • Pas besoin d'installer un serveur.
  • La base est directement intégrée dans le programme.
  • Utilise le langage SQL comme MySQL.

Où on trouve SQLite :

  • Téléphones Android
  • Applications iPhone
  • Navigateur web
  • Logiciels de bureau

Avantages :

  • Très rapide
  • Facile à utiliser
  • Aucune installation
  • Léger

Limites :

  • Pas adapté aux gros sites web
  • Peu d'utilisateurs simultanés

MongoDB

MongoDB est une base NoSQL qui stocke les données sous forme de documents. Chaque document ressemble à un objet JSON utilisé en JavaScript.

Fonctionnement :

  • Les données sont stockées en documents JSON.
  • Pas de tables comme MySQL.
  • Les documents peuvent avoir des structures différentes.
  • Facile à utiliser avec JavaScript.

Utilisations réelles :

  • Applications web modernes
  • API REST
  • Réseaux sociaux
  • Applications temps réel

Avantages :

  • Flexible
  • Très rapide pour les grosses données
  • Facile à faire évoluer

Exemple document MongoDB :

{
  "nom":"Ali",
  "age":25,
  "ville":"Abidjan"
}

Quand choisir chaque base ?

Choisir MySQL :

  • Site web avec beaucoup d'utilisateurs
  • Données bien organisées
  • Système de connexion

Choisir SQLite :

  • Application mobile
  • Petit projet
  • Stockage local

Choisir MongoDB :

  • Applications modernes
  • Données flexibles
  • Projet JavaScript
